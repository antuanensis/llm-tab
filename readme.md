# llm-tab

An AI-powered guitar tab analyzer that turns any ASCII tab into a full music theory lesson. Paste a raw tab, get back chord detection, key analysis, Roman numeral progressions, scale recommendations, practice exercises, and plain-language explanations — all generated by combining rule-based music theory (music21) with Claude as the reasoning and teaching layer.

---

## What it does

Given a raw ASCII guitar tab:

```
e|---3---0---2---0---
B|---0---1---3---0---
G|---0---0---2---0---
D|---0---2---0---2---
A|---2---3---x---2---
E|---3---x---x---0---
  G   C   D  Em
```

The pipeline produces:

| Output | Example |
|---|---|
| **Key detection** | G major (92% confidence) |
| **Chord names** | G major, C major, D major, E minor |
| **Roman numerals** | I → IV → V → vi |
| **Non-diatonic flags** | borrowed chords, secondary dominants |
| **Scale suggestions** | G Ionian, G major pentatonic, D Mixolydian |
| **Guide tones** | target F# and A over the D chord |
| **Beginner summary** | plain-language explanation of how the progression feels |
| **Theory deep dive** | functional harmony, voice leading analysis |
| **Practice exercises** | 3 specific drills tailored to the progression |
| **Cheat sheet** | one-liner summary of key, numerals, and best scale |

## Architecture

```
backend/
├── parser/
│   ├── tuning.py          — open-string MIDI constants (standard, drop-D, open-G)
│   └── tab_parser.py      — regex ASCII tab parser → ParsedChord objects
├── theory/
│   ├── key_detector.py    — music21 Krumhansl-Schmuckler key detection
│   ├── chord_detector.py  — MIDI pitch sets → chord name + Roman numeral
│   └── scale_suggester.py — rule-based scale recommendations per chord
├── llm/
│   ├── prompts.py         — system + user prompt templates for Claude
│   └── client.py          — Anthropic SDK wrapper, returns EnrichedOutput
├── models/
│   ├── tab.py             — TabInput, ParsedNote, ParsedChord
│   ├── analysis.py        — TheoryAnalysis, ChordAnalysis, ScaleSuggestion
│   └── response.py        — EnrichedOutput (full API response shape)
├── api/routes/analyze.py  — POST /api/analyze (orchestrates full pipeline)
└── main.py                — FastAPI app entry point

frontend/
├── src/
│   ├── types/analysis.ts      — TypeScript mirrors of Pydantic models
│   ├── api/analyzeTab.ts      — fetch wrapper for POST /api/analyze
│   ├── components/
│   │   ├── TabInput.tsx        — textarea + submit button
│   │   ├── EnrichedOutput.tsx  — renders all analysis sections
│   │   ├── ChordCard.tsx       — per-chord: name, numeral, scales, guide tones
│   │   ├── PracticeExercises.tsx
│   │   ├── CheatSheet.tsx
│   │   └── LoadingSpinner.tsx
│   ├── App.tsx                 — root component, manages state
│   └── styles/index.css        — dark-theme CSS (no external UI library)
└── vite.config.ts              — proxies /api → localhost:8000
```

**Pipeline flow:**

```
ASCII tab text
    └─▶ tab_parser.parse()         → list[ParsedChord]
    └─▶ key_detector.detect()      → (key, mode, confidence)
    └─▶ chord_detector.detect_all()→ list[ChordAnalysis]  (includes scales + guide tones)
    └─▶ TheoryAnalysis assembled
    └─▶ llm.client.generate()      → EnrichedOutput  (Claude fills in the explanations)
    └─▶ POST /api/analyze response
```

## Setup

### Prerequisites

- Python 3.11+
- Node.js 18+ (via [fnm](https://github.com/Schniz/fnm), nvm, or system install)
- An [Anthropic API key](https://console.anthropic.com/)

### 1. Clone and configure

```bash
git clone <repo>
cd llm-tab
cp .env.example .env
# edit .env and set ANTHROPIC_API_KEY=sk-ant-...
```

### 2. Backend

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3. Frontend

```bash
cd frontend
npm install
```

## Running

Start both servers (two terminals):

**Backend** — from the project root:
```bash
source backend/.venv/bin/activate
python -m uvicorn backend.main:app --reload
# Runs on http://localhost:8000
```

**Frontend:**
```bash
cd frontend
npm run dev
# Runs on http://localhost:5173
```

Open [http://localhost:5173](http://localhost:5173), paste a tab, click **Analyze Tab**.

The Vite dev server proxies `/api/*` requests to `localhost:8000`, so no CORS configuration is needed during development.

## API

### `POST /api/analyze`

**Request body:**

```json
{
  "tab_text": "e|---3---0---2---0---\nB|---0---1---3---0---\n...",
  "tuning": "standard",
  "capo": 0
}
```

| Field | Type | Default | Description |
|---|---|---|---|
| `tab_text` | string | required | Raw ASCII tab (multi-line) |
| `tuning` | string | `"standard"` | `"standard"`, `"drop_d"`, or `"open_g"` |
| `capo` | int | `0` | Capo fret (shifts all MIDI pitches up) |

**Response:**

```json
{
  "theory": {
    "key": "G",
    "mode": "major",
    "key_confidence": 0.92,
    "chords": [
```

## Tab format requirements

The parser handles the standard 6-line ASCII format:

```
e|---frets---
B|---frets---
G|---frets---
D|---frets---
A|---frets---
E|---frets---
```

Supported:
- Multi-digit fret numbers (`10`, `12`, etc.)
- Muted strings (`x` or `X`)
- Open strings (`0`)
- Ornament characters (`h`, `p`, `b`, `/`, `\`) — parsed as spacers, not played
- Optional chord label line below the grid (e.g., `  G   C   D  Em`)
- Capo annotation via the `capo` API field
- Alternate tunings: `drop_d`, `open_g`

Not yet supported:
- Rhythm notation / timing values
- Multi-section tabs (only the first tab grid is parsed)
- Chord diagrams (only tablature lines)

## Configuration

The backend reads from a `.env` file in the project root or from environment variables:

| Variable | Required | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | Yes | Your Anthropic API key |

The LLM model is hardcoded to `claude-sonnet-4-6` in `backend/llm/client.py`. Change it there to use a different model.

## Extending

**Add a new tuning** — add an entry to `backend/parser/tuning.py`:
```python
MY_TUNING = {0: 64, 1: 59, 2: 55, 3: 50, 4: 45, 5: 36}  # low-E = C2
TUNINGS["my_tuning"] = MY_TUNING
```

**Change explanation style** — edit the system prompt in `backend/llm/prompts.py`. The JSON output schema is defined there and must stay in sync with `backend/models/response.py` and `frontend/src/types/analysis.ts`.

**Add scale rules** — extend the `suggest_scales()` function in `backend/theory/scale_suggester.py`. The `_SCALE_INTERVALS` dict at the top covers common modes and pentatonics.

**Add a new frontend section** — add a field to `EnrichedOutput` in `response.py`, include it in the LLM JSON schema in `prompts.py`, mirror it in `frontend/src/types/analysis.ts`, and render it as a new component under `frontend/src/components/`.
